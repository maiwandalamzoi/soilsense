## Variant B — Technical / data science audience

> Sharing a project that's been quietly eating my evenings for the past two weeks.
>
> **SoilSense** — an open-source soil-management dashboard that combines:
> • The SoilGrids 2.0 REST API (250 m global soil properties)
> • Google Earth Engine (MODIS NDVI, CHIRPS rainfall, SRTM, ESA WorldCover)
> • A full Python implementation of RUSLE with Williams/EPIC K-factor and Van der Knijff NDVI→C conversion
> • A Random Forest degradation classifier with local feature attribution
> • A rule-based recommendation engine mapped to FAO VGSSM
> • ReportLab PDF export that actually looks professional
>
> A few design choices I'm proud of:
>
> 1. **Honest ML.** The degradation classifier is trained on synthetic labels generated by a transparent rule-based *teacher* function. The README says so explicitly and the notebook shows exactly how to retrain on real degradation polygons (LADA-L, Trends.Earth, ESA CCI). No performance theater.
>
> 2. **Graceful degradation.** If Earth Engine isn't configured, the app falls back to a country-level rainfall climatology and neutral NDVI prior. The sidebar shows the status. Demos never break.
>
> 3. **Pure-function domain logic.** Everything below the Streamlit layer is plain Python with no UI imports — unit-testable, reusable, portable to any other frontend or batch pipeline.

## Posting strategy

**Day of launch (Variant A):** posts best Tuesday–Thursday, 08:00–10:00 CET for European/African audiences, 10:00–12:00 ET for North American.

**One week later (Variant B):** if engagement is good, amplify to the data science crowd.

**Two weeks later:** a follow-up post with a case study — pick a real location you know well and walk through the output. This is what converts a launch into ongoing visibility.

## What to avoid

- Don't open with "I'm excited/thrilled to announce". Burn-worthy on LinkedIn right now.
- Don't overclaim model accuracy. The "honest ML" angle (Variant B) plays better with sophisticated audiences than inflated numbers.
- Don't bury the demo link. First or second line, always.
- Don't forget the image/video. A short screen-recorded walkthrough (30–60s) outperforms any text-only post.
